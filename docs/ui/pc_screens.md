# 💻 PC側 画面仕様

PCはメインモニターとして機能し、ゲームの進行状況とリッチなビジュアルを表示します。

## 1. ホーム画面
* **プレイ方法の選択:** 「ログインなしで遊ぶ（ゲスト）」ボタンと「ログインして遊ぶ」ボタンを用意し、どちらでも遊べるようにする。
* **ホーム画面:** タイトルロゴとゲームスタートへの導線（ログイン/ゲスト選択）を配置。
* **ログイン（任意）:** ユーザー名を入力してログインすると、過去のスコアや結果が保存されるなどの拡張を行えるようにする。

## 2. コントローラー接続画面
* **目的:** iPhoneをWebSocketの同じルームに参加させる。
* **UI要素:** * 画面中央に大きくQRコードを表示（ルームIDを含むURL）。
  * 「iPhoneでQRを読み取って参戦せよ！」のテキスト。

## 3. 具材選択画面
* **UI要素:** スープに入れたいベースの具材や、難易度（落ちてくるスピード）を選択する画面。
* **[優先順位低] ゲームの説明画面:** 「タイミングを合わせてスマホを突き出せ！」という1枚絵のチュートリアル。
* **[将来構想・優先順位低] 具材ショップ連動:** PC画面を「具材がいっぱい並んだショップ」のようなUIにする。選んだ具材は、iPhone側（トレー）に一覧表示されるような二画面の役割分担を活かしたリッチな見せ方（余裕があったら実装したいアイデア）。

## 4. ゲーム画面（メインプレイ）
* **4-1. 具材をパンチする画面:**
  * **UI要素:** 音ゲー（太鼓の達人など）のように、上から落ちてくる、または左右から流れてくる具材。下部にはグツグツ煮える鍋。
  * **アクション:** ヒット判定ラインに具材が重なった瞬間に、iPhoneから「パンチ信号」を受信すると具材が真っ二つに割れて鍋に入る。
* **4-2. [優先順位低] 具材をかき混ぜる画面:**
  * パンチフェーズ終了後、iPhoneをぐるぐる回す動作でスープを完成させるフェーズ。

## 5. リザルト画面
* **UI要素:**
  * **右上の配置:** ランク（S, A, B, C）と、スコアの具体的な数値。
  * **中央の配置:** 完成したハッピースープのイラスト。
  * **中央右の配置:** 6角形のレーダーチャート（甘味, 塩味, 苦味, 酸味, うまみ, 辛味）。
  * **画面下部の配置:** AI（Gemini等）が生成した、そのスープに対する「一言コメント＆ハッピーポエム」。
