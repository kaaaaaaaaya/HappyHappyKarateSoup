# 📱 iPhone側 画面仕様

iPhoneは画面を注視せず「手元の感覚」で遊ぶためのコントローラーとして機能します。

## 1. PC連携画面
* **UI要素:** 全画面のカメラビュー（この時点では基本**縦持ち**）。
* **アクション:** PC画面のQRコードを読み取ると、自動的にWebSocketサーバーへ接続し、次の画面へ遷移する。

## 1.1. メニュー操作用コントローラー画面
* **基本姿勢:** **縦持ち**。
* **役割:** PC上で表示されている画面（主に具材選択やリザルトからの遷移など）を操作するリモコン。
* **UI要素:** 「上下左右」の十字キー（または方向ボタン）と、「決定（OK）」ボタンを大きく配置。
* **アクション:** ボタンをタップするとPC側へシグナルが送られ、PC画面のカーソル移動や項目の決定が行われる。PC側の具材選択画面などと連動する。

## 1.2. ゲーム準備画面（持ち替え・カウントダウン）
* **UI要素:** 「スマホを反時計回りに90度回してこっち向きに握ってね」という、右利きを想定した持ち替え案内のイラストやアニメーションを表示。
* **アクション:** 縦持ちから横持ちへの移行を促す。PC側と同期して「3, 2, 1」とカウントダウンを表示する（合わせて短い振動などを入れても良い）。

## 2. ゲーム画面
* **基本姿勢:** **横持ち（ランドスケープ）**。
* **UI要素:** * 画面中央に「前へ突き出した拳」のイラスト。
  * [優先順位低] 画面端にポーズ（一時停止）ボタン、またはWiiリモコンのような特定のジェスチャーボタン。
* **アクション（入力）:** 端末の加速度センサーを用いて、前に押し出す（パンチする）動きを検知し、PCへ送信。
* **フィードバック（出力）:**
  * **視覚:** パンチした瞬間に拳の周りにエフェクトが出る。
  * **触覚＆聴覚（重要）:** PC側で具材にヒットしたという信号を受信したら、即座に**Taptic Engineによる強力なバイブレーション**と、**ASMRのような心地よく爽快な打撃音（スパーン！という音や、具材が砕ける音）**を鳴らす。

## 3. リザルト画面
* **UI要素:**
  * 中央のイラストが「前へ突き出した拳」から、「煙をシュウゥゥ…と出して疲労困憊の拳」に切り替わる。
  * 「PCの画面で結果を見よう！」というテキストを表示し、視線をPCへ誘導する。
  * 少し待機した後、自動的に（またはボタンタップで）**「1.5. メニュー操作用コントローラー画面」に戻り**、PC画面の「もう一度遊ぶ」「ホームに戻る」などの操作ができるようにする。
