# 📂 リポジトリ構成 (Repository Structure)

このプロジェクト（HappyHappyKarateSoup）の全体的なフォルダ構造とそれぞれの役割を初学者向けにざっくりと解説します。
チーム開発を進める中でどこに何があるか迷ったときは、このドキュメントを思い出し、道しるべにしてください。

## 📍 フォルダの全体図

```text
HappyHappyKarateSoup/
 ├── apps/         # ユーザーが直接触れる「フロントエンド」のアプリ群
 ├── services/     # 裏側でデータ処理を行う「バックエンド」のシステム群
 ├── docs/         # プロジェクトの仕様書やデザインなどのドキュメント群
 ├── .agent/       # AI開発アシスタント用の設定ルール（AI向けの指示書）
 └── README.md     # はじめに読むべきプロジェクトの概要
```

## 📁 各フォルダの役割

### 1. `apps/` (ユーザーが触れるアプリ)
ユーザーが見たり、操作したりするアプリケーションのコードを置く場所です。
* **`apps/web/`**: PCのモニター側に表示するWebアプリケーション（React + Vite）が入っています。ゲーム画面やリザルト画面はここで作ります。
* **`apps/ios-controller/`**: ユーザーが手に持って振るコントローラーとなるiPhoneアプリ（Swift）が入っています。QRコード読み取りや、パンチ・チョップ動作の判定を行います。

👉 **さらに詳しい構成は [apps.md](./apps/apps.md) を参照**

### 2. `services/` (裏側のシステム)
アプリからのデータを処理したり、スコアを保存したり、AIと通信したりする「サーバー側」のコードを置く場所です。
* **`services/backend/`**: Java (Spring Boot) で書かれたAPIサーバーのコード群が入っています。スコア計算や、具材の落ちてくるタイミング（譜面）の管理など、ゲームのコアとなる仕組みがここにあります。

👉 **さらに詳しい構成は [services.md](./services/services.md) を参照**

### 3. `docs/` (ドキュメント)
コード以外の「このアプリはどう動くべきか」「どんな画面か」を整理している場所です。
* **`docs/ui/`**: 画面の見た目や遷移について（PC側、iPhone側、デザインのルールなどをまとめた場所）。
* **`docs/architecture/`**: 現在見ているこの場所です。システム全体の構造や仕組みをまとめます。
