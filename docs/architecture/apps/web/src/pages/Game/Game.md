# 🎮 `apps/web/src/pages/Game/` フォルダの構成

`pages/Game/` フォルダの中には、PC側Webアプリのメイン機能となる「ゲームプレイ画面」に関連するコードがまとまっています。UI（画面の表示）とLogic（ゲームの進行処理）が分離されているのが特徴です。

## 📍 フォルダ構造

```text
apps/web/src/pages/Game/
 ├── Game.tsx              # ゲーム画面全体のUIコンポーネント
 ├── useGameLogic.ts       # ゲームの進行や状態を管理するカスタムフック
 └── types.ts              # ゲーム内で使用する型定義ファイル (TypeScript)
```

## 各ファイルの詳細

### ⚛️ `Game.tsx` (表示・UI)
Reactのコンポーネントファイルです。主に「画面の描画」と「デザイン・アニメーション」を担当します。
* 3D的に奥から手前に具材（絵文字）が降ってくるCSSアニメーション（`@keyframes`等）の定義。
* `useGameLogic` を呼び出して現在のゲーム状態（カウントダウン中か、どの具材が表示されているか等）を受け取り、それを元に画面要素をレンダリングします。
* 鍋やレーン、UIなどのDOM構造とスタイリングが記述されています。

### ⚙️ `useGameLogic.ts` (状態管理・ロジック)
UIと分離された、ゲームの核となる動き（ロジック）を定義する「カスタムフック」です。
* gcloud storage(GCS)から譜面取得の失敗時、src/main/resources/charts/play/<difficulty>/*.json`から`charData.json` などの譜面データ（具材が落ちてくるタイミングや種類など）を読み込みます。
* 開始前の「カウントダウン処理」と、進行中の状態（`Phase`）の管理。
* 時間経過に応じて出現するべき具材（`activeIngredients`）を計算しリスト化するなどの内部的なゲームループ・タイマー処理を含みます。

### 🏷️ `types.ts` (型定義)
`Game.tsx` と `useGameLogic.ts` の両方で共通して使われる独自の型（データの形）を定義する場所です。
* `Phase`: 現在のゲームの状態が「カウントダウン (`countdown`)」か「プレイ中 (`playing`)」か。
* `ActionType`: アクションの種類が「パンチ (`punch`)」か「チョップ (`chop`)」か。
* `Ingredient`: 落ちてくる具材のデータ構造（id、絵文字、座標、アクションの種類など）。
