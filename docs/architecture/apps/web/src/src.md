# 📁 apps/web/src/ フォルダの構成

`apps/web/src/` 配下は、PCモニター用Webアプリケーション（React）の核となるソースコードやリソースを管理するディレクトリです。

## 📍 `src/` 直下の構造

```text
apps/web/src/
 ├── assets/               # アプリ内で使用する画像やアイコンなどの静的ファイル
 ├── pages/                # 各画面（ページ）を構成するReactコンポーネント
 ├── testdatas/            # 開発・テスト用のモックデータ（JSONなど）
 ├── App.tsx               # アプリ全体のルーティング定義
 ├── App.css               # App.tsx に紐づくスタイルシート
 ├── main.tsx              # Reactアプリケーションのエントリーポイント
 └── index.css             # アプリケーション全体のグローバルなスタイルシート
```

## 各フォルダ・ファイルの詳細

### 📂 `assets/`
画面内で表示する画像ファイル（PNG、SVGなど）を格納します。
* `hero.png`, `react.svg`, `vite.svg` などが含まれています。

### 📂 `pages/`
ユーザーがアクセスする「1つの画面（ページ）」単位のReactコンポーネントを配置します。
* **`Home.tsx`**: ゲームのトップ画面、スタート画面。
* **`Login.tsx`**: ログイン画面（ログインありで遊ぶ場合）。
* **`Connect.tsx`**: iPhoneコントローラーとの接続（QRコード表示など）を行う画面。
* **`SelectIngredient.tsx`**: 具材（難易度やステージ）を選択する画面。
* **[`Game/`](./pages/Game/Game.md)**: 実際のゲームプレイ画面に関する処理やコンポーネント。詳細は専用ドキュメントをご参照ください。
* **`Result.tsx`**: ゲーム終了後のスコアや結果を表示する画面。

### 📂 `testdatas/`
バックエンドが未完成の時や、フロントエンド単体でテスト・デバッグを行いたい時に使用するダミーデータ（モックデータ）を格納します。
* `charData-demo.json`: デモ用の譜面やゲームデータなど。

### 📄 ルートファイル群
* **`main.tsx`**: アプリが起動したときに最初に実行されるファイル。`App` コンポーネントをHTMLのDOMにマウントします。
* **`App.tsx`**: `react-router-dom` などを利用して、URL（ `/`, `/game`, `/result` など）と `pages/` 以下の各画面コンポーネントを結びつけるルーティング（導線設定）を行います。
* **`index.css` & `App.css`**: CSSによるデザイン調整ファイルです。`index.css` はアプリ全体、`App.css` は `App` コンポーネント周辺の基本スタイルを担当します。
